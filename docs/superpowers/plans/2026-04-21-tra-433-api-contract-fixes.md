# TRA-433 API Contract & Spec Fixes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix 7 platform-side API contract/spec defects surfaced by the 2026-04-21 black-box eval: ungrantable `scans:write` in the UI, untyped OpenAPI response schemas, silent boolean-filter coercion, Go layout-string leak in timestamp errors, a `TRA-393` reference baked into the public spec, and `identifiers: null` on the locations list endpoint.

**Architecture:**
- **Frontend:** single-line flip in `ScopeSelector.tsx` to expose the existing `readwrite` level for Scans.
- **Backend:** extend `httputil.ListAllowlist` with typed boolean-filter validation; replace `map[string]any` `@Success` annotations with concrete typed envelopes (`ListAssetsResponse`, `GetAssetResponse`, `ListLocationsResponse`, `GetLocationResponse`, `SaveResponse`); scrub the TRA-393 string from `main.go` swaggo header; static-message the timestamp error in reports; mirror the assets-list identifiers bulk-fetch pattern into locations-list storage.
- **Spec:** regenerate via `just backend api-spec`; verify the diff shows typed `$ref`s and no TRA-393 leak; `just backend api-lint` must still pass.

**Tech Stack:** Go 1.25, chi v5, swaggo 1.16, go-kit's `apispec` post-processor, React 18, Vitest + Testing Library.

**Spec:** [Linear TRA-433](https://linear.app/trakrf/issue/TRA-433)

**Branch:** `miks2u/tra-433-platform-scanswrite-ungrantable-untyped-openapi-schemas` (already checked out).

**Run commands from repo root** using `just backend <cmd>` and `just frontend <cmd>`.

**Out of scope (flagged in PR description):**
- Customer-facing docs (`trakrf-docs` repo): scope table needs `scans:write`; pagination doc should say "400" not "clamped to 200". Separate PR there.
- Linear item 3 ("parameters null on list endpoints") is already resolved — current spec has populated `parameters` arrays for `/api/v1/assets` and `/api/v1/locations`.

```go
// ListAllowlist declares which filter and sort fields the endpoint accepts.
// limit, offset, and sort are always allowed.
//
// BoolFilters is a subset of Filters; values for declared boolean filters are
// validated against the literal strings "true" and "false" (case-sensitive).
// Anything else produces a 400 with detail "<name> must be 'true' or 'false'".
type ListAllowlist struct {
	Filters     []string
	BoolFilters []string
	Sorts       []string
}
```

Then in `ParseListParams`, add a bool-filter set alongside the existing `filterAllow`/`sortAllow`, and validate values in the `default` branch. Replace the existing function body (lines 39-82) with:

```go
func ParseListParams(r *http.Request, allow ListAllowlist) (ListParams, error) {
	out := ListParams{
		Limit:   defaultListLimit,
		Offset:  0,
		Filters: map[string][]string{},
	}

	q := r.URL.Query()
	filterAllow := toSet(allow.Filters)
	boolAllow := toSet(allow.BoolFilters)
	sortAllow := toSet(allow.Sorts)

	for key, values := range q {
		switch key {
		case "limit":
			n, err := strconv.Atoi(values[0])
			if err != nil || n < 1 {
				return out, fmt.Errorf("limit must be a positive integer")
			}
			if n > maxListLimit {
				return out, fmt.Errorf("limit must be ≤ %d", maxListLimit)
			}
			out.Limit = n
		case "offset":
			n, err := strconv.Atoi(values[0])
			if err != nil || n < 0 {
				return out, fmt.Errorf("offset must be a non-negative integer")
			}
			out.Offset = n
		case "sort":
			parsed, err := parseSort(values[0], sortAllow)
			if err != nil {
				return out, err
			}
			out.Sorts = parsed
		default:
			if _, ok := filterAllow[key]; !ok {
				return out, fmt.Errorf("unknown parameter: %s", key)
			}
			if _, isBool := boolAllow[key]; isBool {
				for _, v := range values {
					if v != "true" && v != "false" {
						return out, fmt.Errorf("%s must be 'true' or 'false'", key)
					}
				}
			}
			out.Filters[key] = values
		}
	}

	return out, nil
}
```
