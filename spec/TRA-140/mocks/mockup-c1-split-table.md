## Pros & Cons

### Pros
- Familiar master-detail pattern (email, file browsers)
- See list and detail simultaneously
- No animations/transitions to manage
- Keyboard navigable (accessibility)
- Easy to compare by quick-clicking different assets
- Table format familiar, copy/paste friendly

### Cons
- Less horizontal space for each panel
- May feel cramped on narrow screens
- Two scrollable areas (list + detail)
- Requires responsive breakpoint logic

---

## Technical Notes

- CSS Grid: `grid-template-columns: 1fr 1fr` or `40% 60%`
- Responsive breakpoint: < 768px switches to stacked view
- List uses compact card variant
- Detail panel: fixed header, scrollable history table
- URL state: `?asset=CAM-001&range=7d`
- Keyboard nav: tabindex, arrow key handlers
- Mobile: slide-over or full-page navigation
