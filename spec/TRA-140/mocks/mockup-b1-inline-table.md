## Pros & Cons

### Pros
- Simple table is familiar, easy to scan
- No context switch - history appears in place
- Consistent with existing TrakRF table patterns
- Copy/paste friendly for power users
- Lower implementation complexity

### Cons
- Less visually engaging than timeline
- Harder to spot patterns in movement
- Nested table can feel cramped
- Accordion interaction may be unfamiliar

---

## Technical Notes

- Expand/collapse with CSS transitions (max-height or grid-template-rows)
- History data fetched on demand (lazy load)
- Only one expanded at a time (accordion)
- URL state: `?expanded=CAM-001&range=7d`
- Reuse existing DataTable styles for inner table
- Date range stored in local state per expansion
